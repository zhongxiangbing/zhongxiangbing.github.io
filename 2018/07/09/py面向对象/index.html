面向对象

  • 对象是什么 –>对象是一个模型
    • IT行业中 这就是一个对象
      • C/C++ ,Python,PHP,JAVA 细节
    • 楼房：平房，四合院，公寓
    • Class班级，学生–>小明，小芳
  • 为什么这些可以称作一个模型？
    • 有特征，有共同点
    • 把共同点组合起来就可以抽象总结出一个模型
    • 那么这个模型就可以称作为一个对象

  • ==class用来定义一个类(一个共同点的模型)==

    • 属性
      • 函数：方法，人们喜欢把类中的函数叫方法
      • 变量： 字符串，列表，元组，字典集合…
    • 一个属性的集合
    • 实例：在内存里通过模型实例化创建数据
      • 实例是一个容器一样，包含了所有类里的属性，实际上存在于内存中的
    • 实例化：通过类+括号的方式

  • 面型对象的语法中我们可以通过实例化后的变量去获取相对应的属性(xxxx.xxxx)

  • ==Python中处处接对象==

    self指针的意义、

  • self是一个将我们函数声明为单独为一个实例工作的参数
  • 每增加一个self的函数或者变量都只是单独为自己的实例来工作
    • s1.func() #相当于调用了函数，只不过func(s1)这样调用都史使用了同一个函数
    • s2.func()
    • 当实例调用时,当前实例都会覆盖self(实参传入进去)
    • self用来指向当前工作实例
      • self是约定俗称的命名，你也可以叫别的，但是一般都叫self

        init

面向对象属性

  • 属性

  • ==变量==

    • 实例的变量：只能被实例所访问(私有的)
      • 实例的变量可以被实例所使用
      • s1.name 实例点的方式获取到属于实例的私有变量
        • 属于实例的self.xxx的方式无法通过类去访问，因为这些属性需要实例化才能被访问。
        • self.xxx这些变量都是实例私有的
    • 类的变量：能被实例和类同时访问(共享的)
      • ==但是如果实例修改了或者重新赋值了这个变量将升级成实例私有的。==
      • ==如果这个变量即使没有提前出现过，被实例重新修改了或者赋值，那么也会成为实例独有的。==
    • 内部是私有的，外部是公用的(共享的)
    • 类和实例均支持在创建后，存活期间添加新的属性。
      • 实例添加的时候，如果你职位一个单独实例添加，那么其他的实例不会拥有。
      • 类添加的时候，都会拥有。

        方法(函数)

  • 实例的方法只能被实例所调用，而且默认实例方法第一个位置上的参数，==必须传递调用实例！==

    • def func(self)…
    • xx.func() 来调用一个实例函数，那么实例一定会作为第一个参数传递进来。
    • 函数中的self也可以让我们方便的直接获取到属于实例的私有变量。
    • 实例函数中也可以访问其他实例属性：因为self来指向当前实例的其他属性，self.other

  • 类的方法

    • 类的普通函数 权限完全封闭
    • 只能被类所使用，实例访问不了这种函数

    • 不能访问类的变量及实例变量

    • 类的高级函数 权限展开了类的变量访问权限

      • 实例和类均可以调用
      • 可以访问属于类的变量
      • 依旧不可以访问属于实例的属性
      • 你有了cls形参之后,也就可以互相指向其他类的中高级函数，也就可以调用了
        • 可以在另一个类中高级函数调用其它类 的高级函数

在这里，必须为类的高级函数第一个参数约定俗称为cls形参,该形参用来接收使用的类

  • 类的静态方法:与普通函数相比，开放了实例访问函数的权限(访问类，实例的变量依旧是关闭的)
    • 首先无法访问属于类的变量，没有指向当前使用类的指针
    • 实例和类均可调用
    • 不可访问属于实例的属性
  • 实例之间可以通过静态方法@staticmethod以及高级类方法@classmethod可以进行数据通信
实例的函数(self) 类的普通函数 类的高级函数(cls) 类的静态函数

继承

  • 继承：父类和子类，当子类继承了父类，那么这个子类就会拥有所有父类中的类容

    • 继承之后，当你的子类没有重写父类的init数据初始化函数，那么还会继续默认使用父类的。
    • 子类可以共享父类所有的，并且还可以添加新的
    • 父类不能共享子类拥有的。
    • super():找到当前的父类
      • super().init()

    • 在子类所使用父类属性的时候，一定要确定，在子类中是否保留了下来，而不是因为覆盖重写丢弃了。

多继承

  • 当一个子类，如果继承多个父类

    • 而且多个父类中有重名相同的属性,那么这个子类会选择在继承先写的(就近原则)

    当你在多继承时一定要注意init函数中 如果有些变量没有被继承下来，

    • 其他父类中的init函数因为没有被继承下来，被最近的父类给抢占了
    • 其他父类中其他函数 可能使用到了一些没有被这个子类所继承下来的属性。
    • 首先，如果子类在访问自身没有的属性时，会去父类中找
    • 父类中有重复的，子类会选择最近的
      *init函数由于普遍一个类都会有，所以现在只会继承下来一个init函数(离他最近的父类)
    • 多继承的顺序：就近原则

    • 一件事物可以在不同的情况下具备不同的处理方式。例:电脑

迭代器

  • 迭代器:抽象的数据

    • 迭代器中包含两个内置函数
      • __next__
      • __iter__

  • 可迭代的对象(iterable)

    • 1：都可以通过for循环访问
    • 2：前一个数据可以找到后一个数据
    • 3: 数据维护了之间彼此维持了访问顺序
    • 4：一般由链表构成,或者数组
    • 5：dict、set无序。无法用线性表示其中顺序，树形结构本质是有顺序的
    • 只要有__iter__函数,那么都可以通过iter去初始化成一个迭代器。
  • 访问迭代器(一定是一个抽象的数据)
    • 无法通过工厂函数实际化
    • 可以通过for循环访问,每一次循环提取一个数据
    • next函数可以在每一次调用时返回一个迭代器里的数据
    • 无法重新生产已经生产出来的数据
    • 挤牙膏，访问完了就无法访问了
    • 延时生产数据,啥时候用，啥时候用next或者for循环访问、

无限迭代器

无法被一次性实际化

  • 通过函数自动创建
  • 通过一个序列来访问

有限迭代器

  • iter内置函数直接升华具有iter内置函数的对象

    • iter(list|str|set|tuple|dict|file|sql|range)

  • 模块函数来生产

    • 只要是一个可迭代对象，都可以通过iter变成迭代器，然后用next来访问
    • for：一定访问的是一个可迭代的对象
      • __iter__:检查数据是否是一个可迭代的
      • iter():将迭代对象升华为一个迭代器
      • next():每一次for循环都是调用一次next
      • for循坏的结束其实是捕捉到了StopIteration异常

  • 生成器创建出来的就是可以直接可以用的

  • 元组推导式
